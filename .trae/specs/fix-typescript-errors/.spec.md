# 编辑器核心代码修正 Spec

## Why

当前项目已完成编辑器核心代码开发，但存在 TypeScript 语法错误，影响项目的类型安全和构建。同时需要添加中文注释提升代码可读性，并添加单元测试确保代码质量。

## What Changes

本次修正涵盖 packages/core 整个包的 TypeScript 错误修复、中文注释添加和单元测试编写。

### 发现的 TypeScript 错误（共 7 处，分布在 4 个文件）

#### 1. src/blocks/TableBlockContent/TableBlockContent.ts (1 处)

- 第 43 行：`addNodeView()` 返回的 `LcwDocTableView` 类中 `ignoreMutation` 方法的参数类型错误
    - 问题：`MutationRecord` 应为 `ViewMutationRecord`
    - 原因：`TableView` 基类期望 `ViewMutationRecord` 类型

#### 2. src/editor/LcwDocEditor.ts (2 处)

- 第 1 行：CSS 导入缺少类型声明
    - 问题：`Cannot find module or type declarations for side-effect import of '../style.css'`
    - 解决：添加 CSS 模块声明
- 第 285 行：变量名拼写错误
    - 问题：`lcwDocTipTapOptions` 未定义，应该是 `tiptapOptions`
    - 解决：修正变量名

#### 3. src/editor/LcwDocExtensions.ts (1 处)

- 第 61 行：`types: ['blockContainer']` 赋值类型错误
    - 问题：TypeScript 推断 `types` 为 `never` 类型
    - 原因：空数组 `[]` 无法推断元素类型

#### 4. src/extensions/UniqueID/UniqueID.ts (3 处)

- 第 26 行：`types: []` 初始化为空数组，TypeScript 推断为 `never[]`
- 第 84 行：`filterTransaction.call()` - `this.options` 类型为 `never`
- 第 98、186 行：`types.includes(node.type.name)` - `types` 是 `never[]` 类型
