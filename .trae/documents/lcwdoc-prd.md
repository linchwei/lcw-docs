## 一、产品愿景与定位

### 1.1 产品愿景

打造一个**轻量级、现代化、AI 驱动**的协同文档编辑平台，融合 Notion 的块级编辑体验与飞书的协作能力，为中小团队提供开箱即用的知识管理工具。

### 1.2 目标用户

| 用户类型 | 场景 | 核心诉求 |
|----------|------|----------|
| 个人用户 | 个人笔记、知识管理 | 轻量、美观、AI 辅助写作 |
| 小团队(3-20人) | 团队文档、项目协作 | 实时协作、权限管理、文档关联 |
| 开发者 | 技术文档、API 文档 | Markdown 支持、代码高亮、可扩展 |

### 1.3 竞品参考

| 产品 | 可借鉴点 | 差异化方向 |
|------|----------|------------|
| Notion | 块级编辑、数据库视图 | 更轻量、可私有部署 |
| 飞书文档 | 实时协作、评论 | AI 深度集成 |
| 语雀 | 知识库管理 | 更现代的技术栈、更好的开发者体验 |

### 2.2 安全问题

| 问题 | 严重程度 | 说明 |
|------|----------|------|
| JWT Secret 硬编码 | 🔴 严重 | `secretKey` 写死在代码中 |
| 密码明文存储 | 🔴 严重 | 后端直接存储前端 bcrypt 哈希值，未使用 salt |
| WebSocket 无鉴权 | 🔴 严重 | 协作 WebSocket 无身份验证 |
| Token 传输方式 | 🟡 中等 | 使用自定义 `token` header 而非标准 `Authorization: Bearer` |
| 无速率限制 | 🟡 中等 | API 无请求频率限制 |
| 无 CORS 配置 | 🟡 中等 | 未显式配置跨域策略 |

### 2.3 UX 问题

| 问题 | 说明 |
|------|------|
| 空状态不友好 | 文档列表空状态有引导，但编辑器空白区域无引导 |
| 加载状态缺失 | 多处操作无 loading 反馈 |
| 错误提示不足 | 网络错误、保存失败等无明确提示 |
| 侧边栏不可折叠 | 侧边栏宽度固定，无法折叠/展开 |
| 移动端不适配 | 未做响应式设计 |
| 无快捷键提示 | 编辑器操作无快捷键提示 |

#### F-0201: 安全加固

**优先级**: P0（必须优先完成）

**需求描述**:
修复当前存在的安全隐患，确保基本的安全防护能力。

**功能点**:

| 编号 | 功能 | 详细说明 |
|------|------|----------|
| F-0201-1 | JWT Secret 环境变量化 | 将硬编码的 `secretKey` 改为从环境变量 `JWT_SECRET` 读取，无默认值，启动时校验 |
| F-0201-2 | 密码安全存储 | 后端使用 bcrypt 加盐哈希存储密码，移除前端 bcrypt 加密（改为 HTTPS 传输保护） |
| F-0201-3 | WebSocket 鉴权 | 在 WebSocket 握手时验证 JWT token，拒绝未认证连接 |
| F-0201-4 | 标准 Token 传输 | 改用 `Authorization: Bearer <token>` 标准方式 |
| F-0201-5 | API 速率限制 | 添加 `@nestjs/throttler` 限流（登录 5次/分钟，普通 API 60次/分钟） |
| F-0201-6 | CORS 配置 | 配置允许的域名列表，生产环境禁止 `*` |

**验收标准**:
- [ ] JWT Secret 不再硬编码，缺少环境变量时服务拒绝启动
- [ ] 数据库中密码为 bcrypt 加盐哈希，非明文
- [ ] 未认证 WebSocket 连接被拒绝
- [ ] API 请求使用标准 Authorization header
- [ ] 限流生效，超限返回 429
- [ ] CORS 仅允许配置的域名

#### F-0203: 编辑器输入规则

**优先级**: P1

**需求描述**:
添加 Markdown 风格的快捷输入规则，提升编辑效率。

**功能点**:

| 编号 | 输入规则 | 转换结果 |
|------|----------|----------|
| F-0203-1 | `>` + Space | 引用块 (blockquote) |
| F-0203-2 | `---` / `***` / `___` | 分割线 (divider) |
| F-0203-3 | `#` + Space | H1 标题 |
| F-0203-4 | `##` + Space | H2 标题 |
| F-0203-5 | `###` + Space | H3 标题 |
| F-0203-6 | `-` / `*` + Space | 无序列表 |
| F-0203-7 | `1.` + Space | 有序列表 |
| F-0203-8 | `[]` + Space | 任务列表 |
| F-0203-9 | `` ` `` + Space | 行内代码 |
| F-0203-10 | ` ``` ` + Space | 代码块 |

**实现方式**:
在 `@lcw-doc/core` 包中添加 ProseMirror InputRules，通过 Tiptap 的 `addInputRules` 扩展方法注册。

**验收标准**:
- [ ] 所有输入规则在空行触发时正确转换
- [ ] 转换后光标定位到内容区域
- [ ] 输入规则在已有内容的行末不触发

#### F-0301: 文档分享与权限

**优先级**: P1

**需求描述**:
实现文档分享功能，支持链接分享和权限控制。

**功能点**:

| 编号 | 功能 | 详细说明 |
|------|------|----------|
| F-0301-1 | 分享链接生成 | 生成唯一分享链接，支持复制 |
| F-0301-2 | 访问权限 | 三级权限：仅查看 / 可评论 / 可编辑 |
| F-0301-3 | 链接有效期 | 可设置链接过期时间：永不过期 / 7天 / 30天 |
| F-0301-4 | 密码保护 | 可选设置访问密码 |
| F-0301-5 | 访客模式 | 未登录用户通过分享链接查看文档（只读） |
| F-0301-6 | 分享管理 | 文档所有者可查看/撤销分享链接 |

**数据模型变更**:

```
新增 ShareEntity:
  - id: number (PK)
  - shareId: string (unique, nanoid)
  - pageId: string (FK → PageEntity)
  - permission: enum (view/comment/edit)
  - password: string (nullable, bcrypt)
  - expiresAt: Date (nullable)
  - createdBy: UserEntity (ManyToOne)
  - createdAt: Date
```

**API 变更**:

```
POST   /api/page/:pageId/share       - 创建分享链接
GET    /api/page/:pageId/shares       - 获取分享列表
DELETE /api/share/:shareId            - 撤销分享
GET    /api/share/:shareId            - 访客获取文档（需密码则验证）
```

**验收标准**:
- [ ] 可生成分享链接并复制
- [ ] 三级权限正确生效
- [ ] 链接过期后无法访问
- [ ] 密码保护正常工作
- [ ] 访客可查看文档
- [ ] 分享链接可撤销
