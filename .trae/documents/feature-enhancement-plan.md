# 协同文档功能增强计划

## 当前功能清单

| 类别 | 已实现功能 |
|------|-----------|
| 文档编辑 | BlockNote 编辑器、Markdown 快捷输入、Slash 菜单、@引用文档 |
| 协作 | Yjs 实时协作、WebSocket 同步、PostgreSQL 持久化、多用户光标 |
| AI | 全局 AI 对话、选中文本 AI 菜单（续写/改写/翻译/总结）、行内 AI 面板 |
| 文档管理 | 创建/删除/收藏、回收站、最近编辑、搜索、文档图谱 |
| 导出 | Markdown/HTML/Word/PDF/纯文本 |
| 评论 | 创建/回复/解决/删除评论、评论面板侧栏 |
| 分享 | 分享链接、密码保护、过期时间、权限控制（查看/评论/编辑） |
| 其他 | 文档大纲、模板库、Markdown 上传、字数统计、深色模式、快捷键 |

## 建议增强功能（按优先级排序）

### 🔴 高优先级 — 核心体验提升

#### 1. 文档版本历史
**为什么需要**：协作场景下误操作难以恢复，用户需要回溯和对比历史版本。
- 后端：Yjs 已经通过 PostgreSQL 持久化，可基于 Yjs 的 snapshot 机制定期保存版本快照
- 前端：版本历史面板，支持预览历史版本、与当前版本对比、恢复到指定版本
- 涉及：新增 VersionEntity、VersionService、VersionController；前端新增 VersionPanel 组件

#### 2. 全文搜索（服务端搜索）
**为什么需要**：当前搜索仅匹配文档标题，无法搜索文档内容，对长文档和多文档场景不够用。
- 后端：基于 PostgreSQL 的 `tsvector` 全文搜索，或集成 Meilisearch
- 前端：增强 SearchDialog，支持内容搜索、关键词高亮、搜索结果预览
- 涉及：PageService 新增 search 方法；SearchDialog 重构

#### 3. 文档文件夹/工作区
**为什么需要**：文档数量增多后，扁平列表难以管理，需要分组和组织能力。
- 后端：新增 FolderEntity（id, name, parentId, userId），PageEntity 增加 folderId 字段
- 前端：侧边栏支持文件夹树形结构、拖拽排序、面包屑导航
- 涉及：新增 FolderModule；Aside 组件重构

### 🟡 中优先级 — 协作与效率

#### 4. 协作者管理优化
**为什么需要**：当前协作者通过分享链接进入，无法看到谁在协作、无法管理协作者列表。
- 后端：新增 CollaboratorEntity（userId, pageId, role），ShareService 集成协作者管理
- 前端：协作者列表面板、邀请协作者、角色权限管理（所有者/编辑者/查看者）
- 涉及：新增 CollaboratorModule；SharePopover 增强

#### 5. 文档标签系统
**为什么需要**：比文件夹更灵活的文档分类方式，一个文档可以有多个标签。
- 后端：新增 TagEntity、PageTag 关联表
- 前端：侧边栏标签分组、文档详情页添加/移除标签、按标签筛选
- 涉及：新增 TagModule；Aside 增强

#### 6. 通知系统
**为什么需要**：协作时需要知道谁评论了、谁@了我、文档被修改等。
- 后端：新增 NotificationEntity，评论/分享/协作文档时触发通知
- 前端：通知铃铛、通知列表、未读标记
- 涉及：新增 NotificationModule；Header 新增通知入口

#### 7. AI 功能增强
**为什么需要**：当前 AI 功能较基础，可以更深度集成到编辑流程。
- AI 一键润色/扩写/缩写（当前 SelectionAIMenu 只有4个操作）
- AI 生成文档大纲
- AI 自动标签/摘要
- 对话历史持久化
- 涉及：SelectionAIMenu 增强、GlobalAIChat 增强

### 🟢 低优先级 — 锦上添花

#### 8. 文档封面和图标自定义
**为什么需要**：让文档更有辨识度，类似 Notion 的封面图和图标。
- 前端：文档顶部封面图区域、自定义 emoji/图标上传
- 涉及：PageEntity 新增 coverImage 字段；Doc 页面增强

#### 9. 文档嵌入和 iframe 块
**为什么需要**：支持嵌入外部内容（视频、网页、代码沙盒等）。
- 前端：新增 iframe 块类型、嵌入对话框
- 涉及：BlockNote 自定义块

#### 10. 移动端适配优化
**为什么需要**：当前移动端体验一般，侧边栏和编辑器需要更好的响应式设计。
- 前端：触摸手势、移动端工具栏、响应式布局优化
- 涉及：Doc 页面、Aside 组件响应式重构

#### 11. PWA 支持
**为什么需要**：离线访问、安装到桌面、推送通知。
- 配置：Vite PWA 插件、Service Worker、manifest.json
- 涉及：vite.config.ts 配置

#### 12. 数据导入增强
**为什么需要**：当前只支持 Markdown 上传，用户可能从 Word/Notion/语雀迁移。
- 后端：支持 .docx 文件解析导入
- 前端：MarkdownUploadDialog 增强
- 涉及：UploadModule 增强

## 推荐实施顺序

1. **全文搜索** — 投入小、收益大，直接提升核心体验
2. **文档版本历史** — 协作场景刚需，基于 Yjs snapshot 实现成本可控
3. **文档文件夹/工作区** — 文档管理的基础能力
4. **协作者管理优化** — 完善协作闭环
5. **AI 功能增强** — 差异化竞争力
6. **通知系统** — 提升协作感知
7. 其余按需实施
