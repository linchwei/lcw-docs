# 腾讯云轻量应用服务器部署教程修改计划

## 修改目标
将 `docs/deployment-tencent.md` 从 CVM+轻量双方案并重的结构，改为以轻量应用服务器为主的教程，移除 CVM 相关内容，补充轻量服务器的特有操作路径和注意事项。

## 具体修改步骤

### 1. 第一部分：腾讯云服务器准备
- **1.1 购买与选型**：移除"方案 A：云服务器 CVM"，将"方案 B：轻量应用服务器"提升为主方案，补充轻量服务器购买操作路径
- **1.2 安全组配置**：将"安全组"改为"防火墙"（轻量服务器叫防火墙），操作路径改为"轻量应用服务器 → 防火墙"
- **1.3 公网 IP**：移除 CVM 按带宽/流量计费的说明，轻量自带公网 IP
- **1.4 SSH 密钥**：操作路径改为"轻量应用服务器 → 密钥对"
- **1.5 防火墙配置**：移除 ufw 配置（轻量服务器用控制台防火墙），改为控制台防火墙配置步骤
- **1.6 Docker 镜像加速**：补充轻量服务器内网 DNS 配置（轻量无 VPC，需手动配 DNS）

### 2. 第二部分：项目部署配置
- **7.4 Docker 镜像加速**：移除"CVM 同地域内网拉取"的说明，改为轻量服务器使用公网镜像加速
- 补充轻量服务器内存优化建议（2G 内存需更激进的 Swap 和日志限制）

### 3. 第三部分：GitHub CI/CD 配置
- 基本不变，CI/CD 与服务器类型无关

### 4. 第四部分：部署流程操作指南
- 基本不变

### 5. 第五部分：故障排查
- **16. 轻量服务器特有问题**：从"附录"级别提升为正式章节，补充更多轻量服务器常见问题（流量超限、端口不通等）

### 6. 附录
- **B. 腾讯云 vs 阿里云对比**：移除 CVM 价格行，改为轻量服务器对比
- 移除 TCR 内网拉取说明（轻量无 VPC，无法使用内网 TCR）
